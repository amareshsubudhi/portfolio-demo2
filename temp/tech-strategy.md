The AWS Well-Architected Framework provides proven principles to design and operate secure, high-performing, resilient, and efficient cloud architectures. CROPS captures its five core pillars.

💰
Pillar 1
Cost Optimization
Design architectures that deliver business value at the lowest possible cost.

Pay only for what you use with consumption-based pricing
Continuously measure, monitor, and optimize spend
Use managed services and right-size resources
Optimize over time as usage patterns evolve
🔄
Pillar 2
Reliability
Ensure workloads consistently perform their intended functions.

Design for failure using fault isolation and redundancy
Automatically recover from failures
Scale horizontally to handle demand changes
Continuously test resilience through chaos and recovery testing
⚙️
Pillar 3
Operational Excellence
Run and monitor systems effectively while continuously improving processes.

Automate operations and infrastructure changes
Use metrics, logs, and alarms for observability
Make small, reversible changes
Learn from failures and operational events
🚀
Pillar 4
Performance Efficiency
Use computing resources efficiently to meet system requirements.

Select the right resource types for workloads
Adopt serverless and managed services where possible
Continuously evaluate and evolve architectures
Leverage caching, parallelism, and asynchronous processing
🔒
Pillar 5
Security
Protect data, systems, and assets while delivering business value.

Implement strong identity and access management
Apply defense-in-depth security controls
Encrypt data in transit and at rest
Automate security best practices and continuous monitoring
